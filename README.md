# Ozon Position Tracker

Скрипт для поиска позиции товара в поисковой выдаче Ozon.
Принимает поисковый запрос и SKU товара, возвращает позицию в JSON.

## Требования

- Python 3.11+
- Windows 10/11 или Linux

## Установка

```bash
git clone https://github.com/Bogdanzhf/List_test.git
cd List

py -3.11 -m venv .venv

.\.venv\Scripts\activate

pip install -r requirements.txt

python -m playwright install chromium

copy .env.example .env
```

## Настройка

Файл `.env` (создаётся из `.env.example`):

```env
HEADLESS=false          # false = видимый браузер (рекомендуется)
MAX_PAGES=3             # страниц для проверки (~36 товаров каждая)
DELAY_MIN=1.5           # минимальная пауза между действиями (сек)
DELAY_MAX=3.5           # максимальная пауза между действиями (сек)
MAX_RETRIES=2           # повторов при блокировке на страницу
RETRY_DELAY_MIN=60      # пауза перед retry (сек)
PROXY_SERVER=           # опционально: http://host:port
```

## Запуск

```bash
python script.py --query "нож туристический" --sku 885084802
```

С подробными логами:

```bash
python script.py --query "нож туристический" --sku 885084802 --verbose
```

## Формат вывода

```json
{
  "query": "нож туристический",
  "sku": "885084802",
  "position": 2,
  "page": 1,
  "total_checked": 2,
  "timestamp": "2026-06-05T22:23:25.185418"
}
```

## Тестовые данные

| Запрос | SKU | Ожидаемый результат | Почему выбрал |
|---|---|---|---|
| `нож туристический` | `885084802` | позиция ~2 | Конкурентная ниша, товар в топе |
| `шампунь` | `1162215244` | позиция ~8 | Высокочастотный запрос|
| `сапоги` | `1106452955` | `not_found` | Товар за пределами первых 3 страниц |
| `зубная щетка` | `3460885817` | позиция на стр. 1–3 | Дополнительная проверка |

SKU взяты из URL товаров на Ozon:
`ozon.ru/product/название-товара-**SKU**/`

## Что работает / что нет

### Работает

- Поиск позиции товара на страницах 1–3 выдачи Ozon
- Lazy loading: скролл с ожиданием стабилизации уникальных SKU (~36 на страницу)
- Корректный JSON-вывод по формату задания
- Обработка сценария `not_found`
- Retry при блокировке IP (до 2 повторов, пауза 60–120 сек, новый браузер)
- Fallback-селекторы карточек товаров
- Поддержка прокси через `PROXY_SERVER`
- Ротация User-Agent, viewport и движка браузера (`BROWSER_TYPE`)

### Ограничения

- **HEADLESS=false рекомендуется** — в headless Ozon чаще ставит капчу
- **Без прокси** при частых запусках Ozon временно блокирует IP
- Рекомендуемый интервал между запусками: **3+ минуты**
- При капче скрипт ждёт до 60 сек с проверкой каждые 5 сек (можно прервать Ctrl+C)

## Защита от блокировок (Часть 2)

- Навигация через главную страницу перед первым поиском
- Случайные задержки 1.5–3.5 сек между действиями
- Playwright Stealth + скрытие `navigator.webdriver`
- Ротация User-Agent и viewport при каждом запуске браузера
- Retry с паузой 60–120 сек и перезапуском браузера
- Опциональный прокси через `.env`
