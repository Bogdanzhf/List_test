# Ozon Position Tracker

Скрипт для поиска позиции товара в поисковой выдаче Ozon.
Принимает поисковый запрос и SKU товара, возвращает позицию в JSON.

## Требования

- Python 3.11+
- Windows 10/11 или Linux

## Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Bogdanzhf/List_test.git
cd List

# 2. Создать виртуальное окружение
py -3.11 -m venv .venv

# Windows
.\.venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Установить браузер Playwright
python -m playwright install chromium

# 5. Создать .env файл
copy .env.example .env
```

## Настройка

Файл `.env` (создаётся из `.env.example`):

```env
HEADLESS=false     # false = видимый браузер (рекомендуется)
MAX_PAGES=3        # страниц для проверки (~8-36 товаров каждая)
DELAY_MIN=2.5      # минимальная пауза между действиями (сек)
DELAY_MAX=5.0      # максимальная пауза между действиями (сек)
TIMEOUT_MS=30000   # таймаут ожидания элементов (мс)
```

**Важно:** `HEADLESS=false` обязателен. В headless-режиме
Ozon блокирует запросы через Antibot Captcha.

## Запуск

```bash
python script.py --query "нож туристический" --sku 885084802
```

С подробными логами:

```bash
python script.py --query "нож туристический" --sku 885084802 --verbose
```

## Формат вывода

```json
{
  "query": "нож туристический",
  "sku": "885084802",
  "position": 2,
  "page": 1,
  "total_checked": 2,
  "timestamp": "2026-06-05T22:23:25.185418"
}
```

## Тестовые данные

| Запрос | SKU | Позиция | Почему выбрал |
|---|---|---|---|
| `нож туристический` | `885084802` | 2 | Конкурентная ниша, товар в топе |
| `шампунь` | `1162215244` | 8 | Высокочастотный запрос, FMCG |
| `сапоги` | `1106452955` | not_found | Проверка сценария отсутствия товара на первых 3 страницах |

SKU взяты из URL товаров на Ozon:
`ozon.ru/product/название-товара-**SKU**/`

## Что работает / что нет

### Работает ✅
- Поиск позиции товара на страницах 1-3 выдачи Ozon
- Корректный JSON-вывод по формату задания
- Обработка сценария `not_found`
- Имитация поведения живого пользователя (задержки, скролл, навигация через главную)
- Ротация User-Agent и viewport

### Ограничения ⚠️
- **HEADLESS=false обязателен** — в headless-режиме Ozon ставит капчу
- **~8 товаров на страницу** вместо ~36 из-за lazy loading Ozon
  (товары подгружаются при скролле, текущий скролл не успевает дождаться всех)
- **Без прокси** — при частых запросах (3+ за 5 минут) Ozon временно
  блокирует IP. Рекомендуемый интервал между запусками: 3+ минуты
- При первом запуске может появиться слайдер-капча — решается вручную
  за 5-10 секунд, скрипт ждёт 60 секунд
