# Лог работы над тестовым заданием

## Общий контекст

Задача: написать парсер позиций товаров на Ozon.  
Инструменты: Python 3.11, Playwright, Pydantic, python-dotenv.  
AI-ассистент: Claude Sonnet (claude.ai) — использовал на всех этапах.  

---

## Этап 1 — Выбор инструмента парсинга

**Промт к AI:**
> "Ozon — SPA на Next.js с антиботом. Какой инструмент выбрать для парсинга:
> requests+BS4, Selenium или Playwright? Объясни trade-off."

**Что ответил AI и что взял:**
AI объяснил, что requests+BeautifulSoup отпадает сразу — Ozon рендерит
контент через JavaScript, в HTML без браузера карточек нет. Между Selenium
и Playwright выбрал Playwright: современнее, async из коробки, лучше
поддерживает stealth-патчи через playwright-stealth.

**Решение:** Playwright + playwright-stealth.

---

## Этап 2 — Проблема с окружением (первый тупик)

При попытке запустить `python -m playwright install chromium` получил:  
No module named playwright

Причина: на машине два Python (3.11 и 3.14), pip устанавливал пакеты
в 3.11, а `python` указывал на 3.14. Пакеты не видели друг друга.

**Что сделал:** создал venv через `py -3.11 -m venv .venv`.
Все последующие команды запускал только внутри активированного venv.
Это заняло около 5 минут, но полностью решило проблему изоляции.

## Этап 3 — Antibot Captcha в headless-режиме (второй тупик)

Написал диагностический скрипт `debug_ozon.py` — открывал страницу
поиска напрямую. Результат: `<title>Antibot Captcha</title>`, ноль карточек.

Ozon детектировал headless Chromium до загрузки контента.

**Что попробовал и что не сработало:**
- Прямой переход на страницу поиска → капча мгновенно
- VPN → Ozon блокирует VPN-адреса отдельным редиректом

**Что сработало:**
- `HEADLESS=false` (видимый браузер)
- Навигация через главную страницу перед поиском
- Имитация поведения пользователя: движение мыши, скролл

---

## Этап 4 — Флаги Chromium навредили (третий тупик)

По совету AI добавил в запуск браузера флаги:  
--disable-web-security  
--allow-running-insecure-content  
--ignore-certificate-errors  

Результат стал хуже: Ozon начал отвечать страницей
"Похоже, нет соединения — выключите VPN", хотя VPN выключен.

**Вывод:** эти флаги сами по себе являются сигналом автоматизации.
Ozon их детектирует и блокирует. Убрал все лишние флаги — поведение
улучшилось.

---

## Этап 5 — Race condition при навигации

Ошибка:
Page.evaluate: Execution context was destroyed, most likely
because of a navigation

Происходило когда скрипт пытался вызвать `page.evaluate` (скролл)
в момент, когда страница уже начала переход. 

**Промт к AI:**
> "Получаю Execution context was destroyed при скролле на главной.
> Как защититься?"

**Решение:** обернул все `page.evaluate` в try/except, логируем как
некритичное предупреждение и продолжаем. Навигацию через главную
сделал с `wait_until="networkidle"`.

## Этап 6 — Проблема ленивой загрузки

Страница Ozon должна содержать ~36 товаров, но скрипт находил только 8.
Ozon использует lazy loading — карточки подгружаются при скролле.

Функция `scroll_page_fully` была написана, но срабатывала не полностью:
страница успевала вернуть управление до того как все карточки загружались.

**Что зафиксировали:** 8 SKU на страницу вместо 36. Это ограничение
текущей реализации, описанное в README как известная проблема.

---

## Этап 7 — Блокировка IP при частых запросах

При запуске 3 раз подряд с интервалом 30 секунд Ozon начинал
показывать "Похоже, нет соединения" — фактически временный бан IP.

**Что работает без блокировки:**
- Интервал между запусками от 3 минут
- Один запрос → пауза → следующий запрос

**Что не решается без прокси:**
- Серия из 3+ запросов подряд с одного IP за короткое время

**Вывод для продакшена:** без прокси-пула устойчивость ограничена.
Архитектурное решение описано в `architecture.md`.

## Итог

| Что пробовал | Результат |
|---|---|
| requests + BS4 | Отпал сразу — нет JS рендеринга |
| Playwright headless | Капча мгновенно |
| VPN + Playwright | Ozon блокирует VPN-IP |
| Лишние флаги Chromium | Детектируется как бот |
| HEADLESS=false + главная | Работает, иногда капча вручную |

**Что работает в финале:**
- Видимый браузер, навигация через главную, имитация поведения
- Парсер находит SKU и возвращает позицию корректно
- Ограничение: ~8 товаров на страницу из-за lazy loading,
  без прокси частые запросы вызывают временный бан IP
